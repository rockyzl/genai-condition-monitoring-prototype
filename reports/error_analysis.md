## Where the baseline is weak (honest notes)

- **Cap-dominated tail (systematic under-prediction).** 11 of 100 test units have true RUL above the 125-cycle cap. The model is trained to saturate at 125, so on these units its error is bounded below by `true - 125` and cannot be removed without changing the target definition. This is a floor on error, not the source of the largest residuals: 0 of the top-5 units above are in this tail.
- **Optimistic over-prediction is the actual worst case.** 3 of the top-5 residuals are over-predictions on mid-life units (true RUL below the cap) — the model calls a moderately worn engine nearly healthy. Fleet-wide, mean signed error is +3.2 cycles with 60/100 units over-predicted. Over-prediction is the operationally dangerous direction, so a deployment would pair this model with a conservative risk-band threshold and human review, never the raw point estimate.
- **Short-history units are hard.** 7 units end before cycle 50. With little observed degradation the rolling features are near their healthy baseline, so predictions collapse toward the cap (correlation of |error| with observed history length = -0.10).
- **No uncertainty estimate.** A single RandomForest point prediction carries no calibrated confidence interval. Per-tree spread could give a rough uncertainty band; that is a clear next step and matters for the human-in-the-loop story.
- **Single fault mode, single condition.** FD001 has one operating condition and one fault mode (HPC degradation). These numbers will not transfer to FD002/FD004 (six conditions) without condition-aware normalisation.
